DNS configuration:

 

To implement Domain Name Server (DNS) we will be implementing BIND (Berkeley Internet Name Domain) which is an open-source DNS server software used to resolve domain names to IP addresses and vice versa. Command to used to below to install the BIND server is dnf install bind.

Figure 21: Installing BIND.

 

Configured Forward and Reverse lookup zones inside the named.conf file, more information below:

        Forward lookup zones are defined as:

zone "nspdom44.local" IN {

type master;

file "nspdom44.local.db";

notify NO;

};

        Reverse lookup zones are defined as:

zone "54.168.192.in-addr.arpa" IN {

type master;

file "192.168.54.db";

notify NO;

};

Figure 22: named config file setting up forward and reverse lookup zones.

Below is forward lookup zone file and more information about this file is:

        Defines domain-to-IP mapping (e.g., Projserver → 192.168.54.1).

        SOA Record: Manages zone admin details (e.g., serial, refresh, retry).

        NS Record: Points to the authoritative DNS server (localhost).

        A Records: Maps hostnames to IPs.

        Purpose: Enables human-readable domain access in a network.

Figure 23: forward lookup zone file for nspdom44.local.

 

Below is reverse lookup zone file and more information about this file is:

        Defines IP-to-domain mapping (e.g., 192.168.54.1 → Projserver.nspdom44.local).

        SOA Record: Contains admin details for the reverse zone.

        NS Record: Points to the authoritative DNS server (localhost).

        PTR Records: Maps IPs to hostnames.

        Purpose: Supports reverse DNS queries for logging and authentication.

Figure 23: reverse lookup zone file for nspdom44.local.
